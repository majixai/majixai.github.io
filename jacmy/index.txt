Key Sophistications of the Fan Effect:
 * Expanding Variance (Entropy Modeling): The Python matrix calculates the upper and lower bounds using the mathematical progression variance = (i + 1) * 0.005 * np.linspace(1, 3, len(t_future)). Because linspace(1, 3) increases linearly as t moves forward, the variance limits naturally expand the further into the future the model predicts. Below is the complete, unabridged source for the Cython engine. This is the mathematical bedrock of the Ouroboros Architecture. By using Cython compiler directives, we bypass the Python Global Interpreter Lock (GIL) entirely.
This file allocates contiguous memory arrays and distributes the hyperbolic matrix calculations across all available CPU threads via OpenMP. It calculates both the position curve P(t) and the velocity/momentum gradient P'(t).
# distutils: extra_compile_args=-fopenmp
# distutils: extra_link_args=-fopenmp
# cython: boundscheck=False, wraparound=False, cdivision=True, nonecheck=False

import numpy as np
cimport numpy as cnp
from cython.parallel import prange
from libc.math cimport exp, sinh

cpdef cnp.ndarray[cnp.float64_t, ndim=2] calculate_cosh_matrix(
    double[:] t, 
    double[:] a, 
    double[:] b, 
    double[:] c, 
    double[:] d):
    """
    Computes the Damped Cosh Oscillator position matrix in parallel.
    Formula: P(t) = a * cosh(b(t-c)) + d
    
    Utilizes exponential breakdown (e^x + e^-x)/2 for maximum C-level hardware optimization
    instead of relying on slower Python-level math libraries.
    """
    cdef int i, j
    cdef int n = t.shape[0]
    cdef int m = a.shape[0]
    
    # Pre-allocate contiguous memory view for maximum L1/L2 cache locality
    cdef cnp.ndarray[cnp.float64_t, ndim=2] res = np.zeros((n, m), dtype=np.float64)
    
    # OpenMP Parallel loop bypassing the GIL completely
    for i in prange(n, nogil=True, schedule='static'):
        for j in range(m):
            # Parametric reconstruction of hyperbolic cosine
            res[i, j] = a[j] * (exp(b[j] * (t[i] - c[j])) + exp(-b[j] * (t[i] - c[j]))) / 2.0 + d[j]
            
    return res

cpdef cnp.ndarray[cnp.float64_t, ndim=2] calculate_velocity_matrix(
    double[:] t, 
    double[:] a, 
    double[:] b, 
    double[:] c):
    """
    Computes the First Derivative (Velocity/Momentum) of the Cosh curve.
    Formula: P'(t) = a * b * sinh(b(t-c))
    
    Used strictly by the ParametricBacktester to evaluate hyperbolic momentum 
    and mathematically confirm breakouts bypassing arbitrary threshold indicators.
    """
    cdef int i, j
    cdef int n = t.shape[0]
    cdef int m = a.shape[0]
    
    # Memory allocation for the momentum gradient
    cdef cnp.ndarray[cnp.float64_t, ndim=2] vel = np.zeros((n, m), dtype=np.float64)
    
    for i in prange(n, nogil=True, schedule='static'):
        for j in range(m):
            # Using C-level sinh for maximum throughput on the derivative calculation
            vel[i, j] = a[j] * b[j] * sinh(b[j] * (t[i] - c[j]))
            
    return vel

### File 4 of 4: The Orchestrator (`ouroboros.yml`)

This file is the crowning achievement of the system. It leverages GitHub Actions not as a CI/CD testing pipeline, but as a **distributed, serverless computing cluster**. 

**Key Architectural Expansions:**
*   **The "Cancel-In-Progress" Override:** By strictly disabling cancellation and triggering the cron job every `* * * * *` (1 minute) while passing `--duration 120` to the Python script, we force Runner A and Runner B to interlock. They process overlapping 60-second windows.
*   **Database Survival via Advanced Caching:** SQLite files are transient on ephemeral CI runners. We use `actions/cache@v4` to save the `market_compression.db` at the end of every run and instantly restore it at the beginning of the next. Combined with Python's WAL (Write-Ahead-Log) pragma, multiple overlapping runners can read/write to the cache without throwing SQL Lock Exceptions.
*   **Branch Isolation (gh-pages):** Pushing updates every minute to a `main` branch would result in catastrophic git merge conflicts due to the overlapping runners. Instead, we use the `actions-gh-pages` package. This bypasses git history tracking and forces a strict overwrite of the `output_html` folder directly to an isolated branch, ensuring TradingView always receives the freshest target parameters without breaking the source repo.

```yaml
name: Ouroboros Quantum Overlap Cluster
on:
  schedule:
    - cron: '* * * * *' # Unrelenting minute-by-minute execution
  workflow_dispatch:

# The Concurrency group is the secret to the Overlap.
# Because cancel-in-progress is false, when the 1:01 PM trigger fires, 
# it DOES NOT kill the 1:00 PM trigger. They run simultaneously.
concurrency:
  group: ouroboros-compute-cluster
  cancel-in-progress: false 

jobs:
  heavy_physics_inference:
    name: Cython PINN Inference
    runs-on: ubuntu-latest
    
    steps:
      - name: 1. Checkout Source Code
        uses: actions/checkout@v4

      - name: 2. Setup High-Performance Python Environment
        uses: actions/setup-python@v4
        with:
          python-version: '3.10'
          cache: 'pip'

      - name: 3. Restore Distributed SQLite WAL Database
        # This step pulls the database from the previous minute's execution.
        # It allows the LSTM to have months of historical memory despite ephemeral hardware.
        uses: actions/cache@v3
        with:
          path: market_compression.db
          key: ${{ runner.os }}-market-db-${{ github.run_id }}
          restore-keys: |
            ${{ runner.os }}-market-db-

      - name: 4. Install ML, OpenMP, and Cython Toolchains
        # Installs build-essential to ensure GCC is available for on-the-fly engine.pyx OpenMP compilation
        run: |
          sudo apt-get update
          sudo apt-get install -y build-essential libgomp1
          pip install --upgrade pip
          pip install yfinance tensorflow plotly scipy cython pandas numpy asyncio

      - name: 5. Execute Ouroboros Core Engine (--duration 120)
        # This locks the runner into a 2-minute cycle. 
        # Minute 1: Gathers high-frequency data alongside the previous runner.
        # Minute 2: Takes over ML inference while the previous runner terminates, 
        #           and the NEXT runner boots up to begin gathering data.
        run: |
          python ouroboros.py --duration 120

      - name: 6. Deploy Matrix Outputs to Edge Network (GitHub Pages)
        # We push ONLY the HTML dashboards and the CSV files.
        # We target the gh-pages branch to isolate the deployment from the main codebase.
        # force_orphan ensures we don't build an infinitely growing, massive git history.
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./output_html
          publish_branch: gh-pages
          force_orphan: true
          commit_message: "QUANTUM OVERLAP: Matrix Refresh [Skip CI]"
          
      - name: 7. Vacuum and Optimize Market Database
        # Cleans the WAL logs and compresses the SQLite file before it gets cached for the next runner
        run: |
          sqlite3 market_compression.db "VACUUM;"
```
