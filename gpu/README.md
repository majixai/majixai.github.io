# GPU Processing Module

Central GPU management layer for the **majixai.github.io** repository.  
Any script in any subdirectory can import this package to offload tensor, matrix,
and numerical workloads to the best available hardware backend.

---

## Directory structure

```
gpu/
├── __init__.py           Public API — GPUManager, GPUDispatcher, kernels
├── manager.py            Backend detection, device context, memory info
├── dispatcher.py         Thread-pool task queue with OOM retry
├── config.json           Per-backend tuning parameters
├── last_run_summary.json Auto-generated by the GitHub Actions pipeline
├── kernels/
│   ├── __init__.py
│   ├── tensor_ops.py     HOSVD · Kalman · Monte Carlo · Regime · Haar
│   └── matrix_ops.py     matmul · SVD · PCA · covariance · normalize · softmax
└── README.md             This file
```

---

## Backends

The manager auto-detects and selects the best backend in this priority order:

| Priority | Backend      | Library              | Notes                                    |
|----------|-------------|----------------------|------------------------------------------|
| 1        | `cuda`       | CuPy                 | NVIDIA GPU (CUDA)                        |
| 2        | `mps`        | PyTorch MPS          | Apple Silicon GPU                        |
| 3        | `tensorflow` | TensorFlow           | Any GPU recognised by TF                 |
| 4        | `cpu`        | NumPy                | Always available — automatic fallback    |

Override via `config.json → backends.priority`, or pass `backend="cpu"` to `GPUManager()`.

## Quick start

### From any subdirectory

```python
import sys, os
# Add repo root to path (adjust the number of ".." as needed)
sys.path.insert(0, os.path.join(os.path.dirname(__file__), '..'))

from gpu import GPUManager, GPUDispatcher
import numpy as np

mgr = GPUManager()          # 'cuda', 'mps', 'tensorflow', or 'cpu'
print(mgr.backend)

# Run a single kernel directly
from gpu.kernels import hosvd, monte_carlo

core, sv = hosvd(mgr, feature_matrix=np.random.randn(30, 8), rank=3)
mc = monte_carlo(mgr, mu=0.001, sigma=0.02, steps=20, n_paths=400)
print(mc["mean"], mc["p5"], mc["p95"])
```

### Via the dispatcher (async / queued)

```python
from gpu import GPUManager, GPUDispatcher
import numpy as np

mgr = GPUManager()
with GPUDispatcher(manager=mgr) as d:
    f1 = d.submit("tensor_ops.hosvd",
                  feature_matrix=np.random.randn(30, 8).astype("float32"),
                  rank=3)
    f2 = d.submit("matrix_ops.pca",
                  data=np.random.randn(100, 10).astype("float32"),
                  n_components=5)

    core, sv   = f1.result()
    pcs, expl  = f2.result()
```

## Available kernels

### `tensor_ops`

| Kernel name                   | Function                          | Key kwargs                               |
|-------------------------------|-----------------------------------|------------------------------------------|
| `tensor_ops.hosvd`            | Rank-k truncated HOSVD            | `feature_matrix`, `rank`                 |
| `tensor_ops.kalman_smooth`    | Kalman filter + RTS smoother      | `observations`, `process_noise`, `obs_noise` |
| `tensor_ops.monte_carlo`      | GBM Monte Carlo forecast          | `mu`, `sigma`, `steps`, `n_paths`, `s0`  |
| `tensor_ops.regime_softmax`   | Softmax regime probabilities      | `scores`, `temperature`                  |
| `tensor_ops.haar_wavelet`     | Haar multi-scale decomposition    | `signal`, `levels`                       |

### `matrix_ops`

| Kernel name                   | Function                          | Key kwargs                               |
|-------------------------------|-----------------------------------|------------------------------------------|
| `matrix_ops.matmul`           | Matrix multiply C = A @ B         | `a`, `b`                                 |
| `matrix_ops.svd_truncated`    | Truncated SVD (U, s, Vt)          | `matrix`, `rank`                         |
| `matrix_ops.pca`              | PCA via SVD                       | `data`, `n_components`, `centre`         |
| `matrix_ops.covariance`       | Sample covariance matrix          | `matrix`, `ddof`                         |
| `matrix_ops.normalize`        | Z-score normalisation             | `matrix`, `axis`, `eps`                  |
| `matrix_ops.softmax`          | Row-wise softmax                  | `logits`                                 |

## Configuration

Edit `gpu/config.json` to tune backends, memory limits, thread-pool size, and kernel hyperparameters.

```json
{
  "backends": {
    "priority": ["cuda", "mps", "tensorflow", "cpu"],
    "cuda":  { "memory_fraction": 0.85, "allow_growth": true },
    "tensorflow": { "memory_fraction": 0.85 }
  },
  "task_queue": { "max_workers": 4, "retry_on_oom": true }
}
```

## GitHub Actions

The `gpu_pipeline.yml` workflow:

- **Schedule**: runs daily at 02:00 UTC
- **Manual trigger** (`workflow_dispatch`):  
  - `kernel`: `"all"` (default) or any single kernel name  
  - `backend`: `"auto"` (default) or override

The workflow writes `gpu/last_run_summary.json` after each successful run.

## Registering custom kernels

```python
from gpu.dispatcher import GPUDispatcher

@GPUDispatcher.register("my_module.my_kernel")
def my_kernel(manager, data, threshold=0.5):
    xp = manager.xp
    arr = xp.asarray(data)
    return manager.to_host(arr[arr > threshold])

# Now dispatchable from anywhere
dispatcher.submit("my_module.my_kernel", data=my_data, threshold=0.3)
```
